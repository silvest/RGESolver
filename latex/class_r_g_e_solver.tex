\doxysubsection{Detailed Description}
A class that performs renormalization group evolution in the context of the SMEFT. 

The class solves the Renormalization Group Equations (RGEs) numerically. A faster, approximate solution that neglects the scale dependence of the anomalous dimension matrix is also available. Only operators up to dimension six that preserve lepton and baryon numbers are considered. The operator basis is the Warsaw basis, defined in \href{https://arxiv.org/abs/1008.4884}{\texttt{ https\+://arxiv.\+org/abs/1008.\+4884}}. {\ttfamily \mbox{\hyperlink{class_r_g_e_solver}{RGESolver}}} splits real and imaginary part of each complex parameter. ~\newline
 ~\newline
 The numerical integration is performed with an adaptive step-\/size routine (the explicit embedded Runge-\/\+Kutta-\/\+Fehlberg method), using the tools in the GNU Scientific Library. See \href{https://www.gnu.org/software/gsl/doc/html/ode-initval.html}{\texttt{ https\+://www.\+gnu.\+org/software/gsl/doc/html/ode-\/initval.\+html}} for all the details. ~\newline
The accuracy level of the numerical integration can be tuned selecting the parameters $\epsilon_{rel}$ and $\epsilon_{abs}$ using the dedicated setter functions. ~\newline
 ~\newline
 All the SMEFT coefficients are set using the \mbox{\hyperlink{class_r_g_e_solver_adc2b8a9a10216ec5285569c8f24cbed2}{Set\+Coefficient}} methods and accessed with the \mbox{\hyperlink{class_r_g_e_solver_a23d5d847021421bc8a9ba03412d2674e}{Get\+Coefficient}} methods. There exist three different signatures for each method, depending on the number of flavour indices of the parameter (0,2,4). ~\newline
 These two routines must be used also for the SM parameters $g_1,g_2,g_3,\lambda,m_h^2,$ $\mathrm{Re}(\mathcal{Y}_u),\mathrm{Im}(\mathcal{Y}_u),$ $\mathrm{Re}(\mathcal{Y}_d),\mathrm{Im}(\mathcal{Y}_d),$ $\mathrm{Re}(\mathcal{Y}_e),\mathrm{Im}(\mathcal{Y}_e)$ (we follow \href{https://arxiv.org/abs/1308.2627}{\texttt{ https\+://arxiv.\+org/abs/1308.\+2627}} for what concerns the conventions in the Higgs\textquotesingle{} sector). ~\newline
 The routines \mbox{\hyperlink{class_r_g_e_solver_a1bdc3a8209979b9f4c583b47fc4988d4}{Get\+CKMAngle}}, \mbox{\hyperlink{class_r_g_e_solver_a891564c063a942fbd01830b30fa8e6e1}{Get\+CKMPhase}}, \mbox{\hyperlink{class_r_g_e_solver_a67f092224a036208d5904da39726ce80}{Get\+CKMReal\+Part}}, \mbox{\hyperlink{class_r_g_e_solver_a9a3972af1912ae6ffebc806ffe05daeb}{Get\+CKMImag\+Part}} should be used when interested in the CKM parameters or elements. The usage of this method is recommended after methods such \mbox{\hyperlink{class_r_g_e_solver_af4ed89d1098aa91112cf9e5a39dd8e7f}{Generate\+SMInitial\+Conditions}} or \mbox{\hyperlink{class_r_g_e_solver_a300d0d8643d30cdb49e0df0294793ba2}{Evolve\+To\+Basis}} that choose a specific flavour basis (\char`\"{}\+UP\char`\"{} or \char`\"{}\+DOWN\char`\"{}), in which cases the CKM matrix is updated. A complete list of the keys that must be used to correctly invoke setter/getter methods are given in tables \doxytablelink{class_r_g_e_solver_SM}{SM}, \doxytablelink{class_r_g_e_solver_0F}{0F}, \doxytablelink{class_r_g_e_solver_2F}{2F} and \doxytablelink{class_r_g_e_solver_4F}{4F}. ~\newline
A summary of the operators symmetry classes is given in table \doxytablelink{class_r_g_e_solver_Sym}{Sym}. ~\newline
 ~\newline
 We follow \href{http://www.utfit.org/UTfit/Formalism}{\texttt{ http\+://www.\+utfit.\+org/\+UTfit/\+Formalism}} for what concerns the conventions for the CKM matrix.
